# Демо-версия хелпдеска (без бэкенда)

Полностью автономная демонстрация веб-панели для показа заказчикам:
**один HTML-файл**, который открывается двойным кликом. Не нужны n8n, Redis,
PostgreSQL, Qdrant, Python — вообще ничего разворачивать не надо.

## Как показать заказчику

Отправьте файл **`dist/vpn-helpdesk-demo.html`** (почта, Telegram, флешка) —
получатель просто открывает его в браузере.

- Вход — **любые логин и пароль** (если ввести `@maria_iv` — войдёте агентом,
  иначе — админом `Алексей Петров` со всеми разделами).
- Все разделы кликабельны: диалоги, ответы клиентам, статусы тикетов,
  биллинг-действия, статистика, настройки, шаблоны, база знаний, операторы.
- Демо «живое»: клиенты отвечают на сообщения оператора, через ~25 секунд
  возвращается клиент из ожидания, через ~45 секунд приходит новое обращение
  (ему отвечает ИИ), ещё через полминуты клиент вызывает оператора —
  срабатывает уведомление на колокольчике.
- Данные живут в памяти вкладки и сбрасываются при обновлении страницы.

## Как это устроено

Используется **тот же фронтенд, что и в бою** (`app/static/*.jsx`) — ни одна
строка боевого кода не менялась. Демо-слой подменяет в браузере
`fetch("/api/...")` и `WebSocket("/ws")` на мок с заготовленными данными:

- `demo-data.js` — демо-данные: диалоги, сообщения, операторы, шаблоны,
  база знаний, статистика, серверы (временные метки считаются от текущего
  момента, поэтому демо всегда выглядит «свежим»);
- `demo-mock.js` — мок всех API-эндпоинтов + фейковый WebSocket + сценарий
  «живых» событий;
- `build.py` — сборщик: инлайнит React, Babel, lottie-player (из npm),
  генерирует статический Tailwind CSS по JSX-исходникам и вшивает всё в один
  файл `dist/vpn-helpdesk-demo.html`.

## Пересборка (после изменений фронтенда или демо-данных)

```bash
python3 demo/build.py
```

Нужны `python3` и `node`/`npx` (для Tailwind CLI). Вендорные библиотеки
кешируются в `demo/vendor/` — сеть нужна только при первой сборке.

Если боевой `app/static/index.html` изменит набор подключаемых скриптов,
сборщик упадёт с понятной ошибкой — обновите список в `build.py`.
